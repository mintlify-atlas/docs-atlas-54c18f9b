FlyDSL lets you write high-performance GPU kernels in Python using a layout algebra inspired by CuTe/CUTLASS, then compiles them end-to-end through MLIR to AMD GPU binaries. You get explicit control over tiling hierarchies, data movement, and instruction scheduling — without leaving Python.

## What is FlyDSL?

FlyDSL is the Python front-end of the **Fly** project — a Flexible Layout Python DSL for AMD ROCm GPUs. It exposes:

- **`@flyc.kernel`** — defines a GPU kernel function traced to MLIR
- **`@flyc.jit`** — JIT-compiles and launches the host-side driver
- **`flydsl.expr` (`fx`)** — layout algebra, copy atoms, MMA atoms, arithmetic, and GPU intrinsics
- **Pre-built kernels** — production-ready GEMM, normalization, attention, and MoE kernels

The compiler lowers Python-traced MLIR through the Fly dialect to ROCDL and binary fatbin artifacts, with automatic disk caching for fast subsequent launches.

## Get started

<Steps>
  <Step title="Build LLVM/MLIR">
    Run `bash scripts/build_llvm.sh -j64` to compile LLVM and MLIR from source (one-time, ~30 min).
  </Step>
  <Step title="Build FlyDSL">
    Run `bash scripts/build.sh -j64` to compile the Fly dialect and Python bindings.
  </Step>
  <Step title="Install in dev mode">
    Run `pip install -e .` for an editable install that reflects Python source changes immediately.
  </Step>
  <Step title="Run your first kernel">
    Follow the [Quickstart](/quickstart) to write a vector addition kernel with layout algebra.
  </Step>
</Steps>
