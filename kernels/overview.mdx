The `kernels/` package ships a collection of production-ready GPU kernels for AMD ROCm hardware. Each kernel is implemented using the `@flyc.kernel` / `@flyc.jit` API from `flydsl.compiler` and can be imported directly into your Python code with no build step beyond installing FlyDSL itself. The kernels target CDNA3 (MI300X, gfx942), CDNA4 (MI350/MI355X, gfx950), RDNA4 (gfx1201), and gfx1250 (MI450) GPU families.

<Note>
  The `kernels/` directory is a Python package. All imports use the `kernels.` prefix, for example `from kernels.layernorm_kernel import build_layernorm_module`. Run with `PYTHONPATH=./` when invoking from the repository root without an editable install.
</Note>

## How to import and use

Every kernel follows a two-step pattern: build a launcher (JIT compile), then call it with PyTorch tensors.

```python
import torch
from kernels.layernorm_kernel import build_layernorm_module

# Step 1: build the launcher (compiled on first call)
launcher = build_layernorm_module(M=32768, N=8192, dtype_str="bf16")

# Step 2: prepare inputs
x      = torch.randn(32768, 8192, dtype=torch.bfloat16, device="cuda")
gamma  = torch.ones(8192,        dtype=torch.bfloat16, device="cuda")
beta   = torch.zeros(8192,       dtype=torch.bfloat16, device="cuda")
output = torch.empty_like(x)

# Step 3: call the launcher
m_in = torch.tensor(32768, dtype=torch.int32)
launcher(x, gamma, beta, output, m_in)
torch.cuda.synchronize()
```

The pattern is the same for every normalization and GEMM kernel — call the builder once per shape/dtype configuration, reuse the returned launcher across calls.

### Preshuffle GEMM quick start

```python
from kernels.preshuffle_gemm import compile_preshuffle_gemm_a8

launch_fn = compile_preshuffle_gemm_a8(
    M=0, N=5120, K=8192,
    tile_m=16, tile_n=128, tile_k=256,
    in_dtype="fp8",
    lds_stage=2,
)

# launch_fn(arg_c, arg_a, arg_b, arg_scale_a, arg_scale_b, M, N, stream)
```

## Choosing the right kernel

Use this decision tree to select the appropriate kernel for your workload.

```
What operation?
│
├── Matrix multiply (GEMM)
│   ├── FP8 / INT8 / INT4 / FP16 / BF16 / FP4  → preshuffle_gemm
│   ├── FP16 or BF16 with split-K reduction     → hgemm_splitk
│   └── Mixture of Experts (gate+up+reduce)     → moe_gemm_2stage / moe_blockscale_2stage
│
├── Normalization
│   ├── With learnable bias (beta)  → layernorm_kernel
│   └── No bias term                → rmsnorm_kernel
│
├── Softmax (row-wise)  → softmax_kernel
│
├── Attention
│   ├── Full-sequence forward pass  → flash_attn_func   (WIP: perf tuning)
│   └── Decode with paged KV cache  → pa_decode_fp8     (WIP: perf tuning)
│
└── Auxiliary
    ├── Fused RoPE + KV cache write  → fused_rope_cache_kernel
    └── Multi-GPU sum reduction      → custom_all_reduce
```

## Shared utilities

All normalization and attention kernels share helper code in `kernels/kernels_common.py` (warp and block reductions), `kernels/mfma_epilogues.py` (MFMA output strategies), and `kernels/mfma_preshuffle_pipeline.py` (preshuffle data movement and layout helpers). You generally do not need to import these directly — they are used internally by the builders.
